# Case Study Skill Mining

Use this skill to convert founder stories into reusable startup guidance. The output is not an article archive; it is a set of evidence-backed patterns that can guide future products.

## Core Rule

Preserve source links and claims, but commit only patterns, summaries, and skills. Do not republish full copyrighted articles.

## Workflow

1. Collect case URLs from allowed public sources or a user-provided URL list.
2. Extract structured signals: user, pain, validation, acquisition, product shape, pricing, revenue, constraints, turning points.
3. Cluster repeated signals across cases. Treat one-off success stories as weak evidence.
4. Convert repeated signals into conditional rules: when a new project has this context, try this move.
5. Generate or update a skill such as `founder-case-patterns`.
6. Record evidence links and confidence, then apply the skill to current startup decisions.

## Evidence Schema

- case title and source URL;
- founder or company if public;
- target user and painful situation;
- first validation move;
- first acquisition channel;
- monetization model and price signal;
- revenue claim if present;
- strongest lesson;
- caveats or reasons it may not generalize.

## Quality Bar

- At least 5 cases for a tentative pattern.
- At least 10-20 cases for a stronger playbook rule.
- A pattern must say when it applies and when it should not be copied.
- Prefer exact source links over memory.
- Keep confidence labels: weak, moderate, strong.

## Tooling

If this repository is available, use `tools/case-study-skill-miner` for repeatable extraction and skill synthesis.
