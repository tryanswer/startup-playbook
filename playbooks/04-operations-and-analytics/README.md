# Operations And Analytics

运营的目标是让产品学习速度变快。每个动作都要能回答：用户是否更快理解、更愿意尝试、更愿意付费、更愿意回来。

## Core Metrics

- Acquisition: 展示、点击、来源、关键词、国家、语言。
- Activation: 注册、上传、生成、完成核心任务。
- Revenue: 支付点击、购买、退款、复购。
- Retention: 次日/7日/30日回访、报告重复查看、功能复用。
- Feedback: 用户原话、投诉、阻塞点、愿望清单。

## Guides

- `ga4-event-tracking.md`: GA4 埋点规范——按 AARRR 分层的事件定义、参数、实现示例和调试方法。上线前必须完成。
- `retention-decision-tree.md`: 留存分析决策树——D1/D7/D30 各阶段的阈值、诊断路径和行动建议。

## Weekly Review

1. 打开 Looker Studio 周复盘 dashboard（见 `templates/weekly-review-dashboard.md`）。
2. 复盘本周最大的增长机会和最大阻塞点。
3. 对照用户原话检查文案和产品是否偏离。
4. 看一个主指标，不同时追太多目标。
5. 检查留存趋势，用 `retention-decision-tree.md` 做诊断。
6. 决定下周实验：继续、停止、调整。
7. 把结论写回 case study，形成下次可复用资产。
