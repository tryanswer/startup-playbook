# Operations And Analytics

The goal of operations is to accelerate product learning speed. Every action should answer: Do users understand faster, are they more willing to try, more willing to pay, and more willing to come back.

## Core Metrics

- Acquisition: Impressions, clicks, sources, keywords, countries, languages.
- Activation: Sign-ups, uploads, generations, completion of core tasks.
- Revenue: Payment clicks, purchases, refunds, repurchases.
- Retention: Next-day/7-day/30-day return visits, report re-views, feature reuse.
- Feedback: User quotes, complaints, blockers, wish lists.

## Guides

- `ga4-event-tracking.md`: GA4 tracking specifications—AARRR-layered event definitions, parameters, implementation examples, and debugging methods. Must complete before launch.
- `retention-decision-tree.md`: Retention analysis decision tree—thresholds, diagnostic paths, and action recommendations for D1/D7/D30 stages.

## Weekly Review

1. Open the Looker Studio weekly review dashboard (see `templates/weekly-review-dashboard.md`).
2. Review this week's biggest growth opportunity and biggest blocker.
3. Check against user quotes to see if copy and product have deviated.
4. Focus on one main metric, don't chase too many goals at once.
5. Check retention trends, use `retention-decision-tree.md` for diagnosis.
6. Decide next week's experiments: continue, stop, adjust.
7. Write conclusions back into case studies, forming reusable assets for next time.
